2.관계연산자
	1)조건을 설정하기 위해 사용한다
	a > b	a가 b보다 큰 조건
	a < b	a가  b보다 작은 조건
	a >= b 	a가 b보다 크거나 같은 조건
	a <= b 	a가 b보다 작거나 같은 조건
	a = b	a가 b랑 같은 조건
	a <> b	a가 b랑 다른 조건
	   !=도 가능

5.SQL 연산자
 5-1조건을 설정할 때 사용하는 연산자
	1)a BETWEEN b AND c
		-> a가 b와 c사이인 조건 (b 와 c는 포함이 됨 : b이상 c이하)  
		->10 BETWEEN 5 AND 100
		->salary BETWEEN employee_id AND 100  (컬럼도 사용 가능)

	2)a IN(b,c,d,...)
		a가 b랑 같거나 a가 c랑 같거나 a가 d랑 같거나 ...
		을 만족하는 조건
	3)a LIKE 패턴
		a가 패턴을 만족하는 조건
	
	4)a 관계연산자 ANY (b,c,d,...)	
	    a          >         ANY (5,6,7)
		-> a관계연산자 b혹은 a관계연산자 c 혹은 a관계연산자 d... 를 만족하는 조건
	
	5)a 관계연산자 ALL (b,c,d,...)
	    a         =          ALL(10,20,30...)
	   a  관계연산자 b 그리고 a 관계연산자 c 그리고 a 관계연산자 d... 를 만족하는 조건
	  a=10 그리고 a=20 그리고 a=30 을 만족하는 조건
	    
      
	6)EXISTS() (서브쿼리 배울때 설명예정)

	7)NOT EXISTS () (서브쿼리 배울때 설명예정)

	8)IS NULL

7.논리연산자
 7-1여러개의 조건을 동시에 적용하기 위해서 사용한다
	조건1 AND 조건2
		조건1 그리고 조건2를 만족하는 조건
	조건1 OR 조건2
		조건1 혹은 조건2를 만족하는 조건
	NOT 조건	
		조건을 만족하지 않는 조건

7-2 조건의 우선순의
	NOT > AND > OR
	우선순위를 바꾸고 싶다면 ()를 쳐줘야함

8. 컬럼 별칭 설정
	SELECT 컬럼명 AS "별칭"
	
	1) AS 생략가능
	2)쌍따옴표(" ") 생략가능
	3)별칭에 띄어쓰기가 있는 경우 쌍따옴표(" ") 생략 불가능
	4)예약어를 별칭으로 사용할 경우 쌍따옴표(" ") 생략 불가능
	5)WHERE절에는 별칭 사용 불가능
	6)ORDER BY절에는 별칭 사용 가능
